Angular Directives Overview

Angular Directives Overview

Directives are markers on DOM element that tell Angular to attach a specified behavior to an existing element.
Angular provides a number of built-in directives, which are attributes we can add to our template elements in order to attach to them a dynamic behaviour.
Angular includes several structural directives:
NgIf
NgFor
NgSwitch, NgSwitchWhen, NgSwitchDefault
And several non-structural directives:
NgClass
NgStyle
NgControlName
NgModel

Structural Directives

Structural Directives

Overview

They can manipulate the DOM structure.
With them we can add, remove or modify DOM elements.
An asterisk (*) precedes the structural directive name
You may apply only one structural directive to an element.

Overview

Asterisk symbol is a "syntactic sugar".
Internally, Angular translates the *ngSomething attribute into a <ng-template> element, wrapped around the host element

				<h2 *ngIf="true">shown</h2>
			

				
					<h2>shown</h2>
The translation is according to the Angular's microsyntax.

More on property bindings ([ngif]="") will be discussed in "Data and Events Bindings" theme.

NgIf


			<tag *ngIf="condition"></tag>
With ngIf directive we can display or hide an element based on a condition.
condition is the result of an expression that we pass into directive.
if condition == true : the element will be included in the DOM
if condition == false : the element will be removed (not just hide) from the DOM

NgIf - examples


			import { Component } from '@angular/core';

			@Component({
				selector: 'app-directives-test',
				styles: [`section>h2{
										border: .1em solid #000; margin: 1em;
								}`],
				template: `<section>
										<h2 *ngIf="false">*ngIf="false"</h2>
										<h2 *ngIf="true">*ngIf="true"</h2>
										<h2 *ngIf="3>2-1">*ngIf="3>2-1"</h2>
										<h2 *ngIf="showH2">*ngIf="showH2"</h2>
									</section>`
			})
			export class DirectivesTestComponent {

				showH2 = 'true';

			}

ngIf; then; else - syntax

Note that we have to use template reference variable in order to refer to blocks in then and else clauses.

NgSwitch

To replace multiple alternative ngIf directives

			<div class="container">
				<div *ngIf="myVar == 'A'">Var is A</div>
				<div *ngIf="myVar == 'B'">Var is B</div>
				<div *ngIf="myVar != 'A' && myVar != 'B'">Var is neither A nor B</div>
			</div>
		

			<div class="container" [ngSwitch]="myVar">
				<div *ngSwitchCase="'A'">Var is A</div>
				<div *ngSwitchCase="'B'">Var is B</div>
				<div *ngSwitchDefault>Var is something else</div>
			</div>

ngFor

Allows to repeat a given DOM element and pass an element of the array (iterable object) on each iteration.
ndFor is useful for generating repeating content, such as a list items, dropdown options, and so on.

			<ul>
				<li *ngFor="let item of [1,2,3]">item {{item}}</li>
			</ul>
		

			. item 1
			. item 2
			. item 3

ngFor - local variables

Inside *ngFor microsyntas, angular provides some local variables useful in our loops, like index, count and etc.
Reference: *ngFor - Local Variables

			<li *ngFor='let item of [1,2,3]; index as i; odd as isOdd'>
				<span>item: {{item}}, index: {{i}}, odd: {{isOdd}}</span>
			</li>
		

			item: 1, index: 0, odd: false
			item: 2, index: 1, odd: true
			item: 3, index: 2, odd: false

ngClass - overview

Adds and removes CSS classes on an HTML element.
Used when we need to conditionally apply multiple stiles to our templates
More powerful than using [className] property binding (discussed in next lectures).

Custom Directives

Custom Directives

Overview

Directives are Components without a view/UI.
Directives do not need the "template" and "style" annotations.
We create directives by annotating a class with the @Directive decorator.

			import { Directive } from '@angular/core';

			@Directive({
				selector: '[selector'
			})
			export class SomeNameDirective {

				constructor() { }

			}

Generate directive

Using Angular CLI:

				ng g d directives/changeContent
directives are not created in separate folder, by default.
Directives must be declared in Angular Modules in the same manner as components.

			import { Directive } from '@angular/core';

			@Directive({
				selector: '[appCustomDirective]'
			})
			export class CustomDirectiveDirective {

				constructor() { }

			}

ElementRef and Renderer2 objects

Angular provides us with two very useful objects: the ElementRef and the Renderer.
The ElementRef object gives us direct access to the DOM element for the directive through the nativeElement property.
The Renderer2 object gives us many helper methods to enable us to modify the DOM element even on Native Aps
Both objects can be injected into a class

ElementRef - example


			ng g d directives/changeContent
		

			import { Directive,ElementRef } from '@angular/core';

			@Directive({
				selector: '[changeContent]'
			})
			export class ChangeContentDirective {

				constructor(private Element: ElementRef) {
						console.dir(Element);
						Element.nativeElement.innerText="Changed by changeContent Directive. ";
				}

			}

Angular Pipes Overview!

Angular Pipes Overview!

AngularJS (Angular 1.X) - uses filters to transform data
In Angular (Angular2+) the filters are now called pipes
Most of AngularJS filters have their Angular pipes equivalents
There is no equivalent for filtering or sorting lists No FilterPipe or OrderByPipe @angular docs

How to use Pipes

How to use Pipes

A pipe takes in data as input and transforms it to a desired output
Angular includes several pipes to add to your template.
There is no need to import them or add them as directives or anything—just start using them.
A pipe is denoted with the pipe symbol : |

Syntax


			{{ data | pipe }}

Built-in Pipes

Built-in Pipes

lowercase

Transforms a string into a lower-case string


			{{ 'string' | lowercase }}
		

			
Lowercase: {{'Ada Byron' | lowercase}}

		

			Lowercase: ada byron

uppercase

Transforms a string into a upper-case string


			{{ 'string' | uppercase }}
		

			
Uppercase: {{'Ada Byron' | uppercase}}

		

			Uppercase: ADA BYRON

currency

Adds currency info to a number


			{{ value | currency [ : currencyCode [ : display [ : digitsInfo [ : locale ] ] ] ] }}
		

			
Currency: {{ 23.55 | currency }}

		

			Currency: $23.55

More on CurrencyPipe @angular docs

percent

Transforms a number into a percent


			{{ 'string' | percent }}
		

			
Percentage: {{ 0.5 | percent }}

		

			Percentage: 50%

date

Transforms a datetime object or string representing a date into a date formated string
DatePipe @angular docs

			{{ value | date [ : format [ : timezone [ : locale ] ] ] }}
		

			
Date: {{ '2018-01-15T09:03:01+0900' | date }}

			
Date: {{ '2018-01-15T09:03:01+0900' | date:'y MMMM - EEEE d' }}

		

			Date: Jan 15, 2018

			Date: 2018 January - Monday 15

more built-in pipes demos


			import { Component, OnInit } from '@angular/core';

			@Component({
				selector: 'app-built-in-pipes',
				template: `

Current time piped into date:'HH:mm:ss' is: {{ now | date:'HH:mm:ss' }}

				styles: [
				]
			})
			export class BuiltInPipesComponent {

				now = new Date();
			}

Custom Pipes

Custom Pipes

Create custom pipes

You can use the CLI:

				ng generate pipe pipeName
Or, you can create your own class with @Pipe annotation and:
the class must implement the PipeTransform interface
imports the Pipe and PipeTransform

example


			import { Pipe, PipeTransform } from '@angular/core';

			@Pipe({
				name: 'fancyQuotes', pure:false
			})
			export class FancyQuotesPipe implements PipeTransform {
				transform(value: any): string {
					return `«${value}»`;
				}
			}
		

			template: `
				

					{{'ada'|fancyQuotes}}
				

			`,
